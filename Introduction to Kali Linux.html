Kali Linux Fundamentals

What is Kali Linux?

Kali Linux is a Debian-based Linux distribution specifically designed for digital forensics and penetration testing. It comes pre-installed with over 600 security and forensics tools, making it the go-to platform for security professionals and ethical hackers.

History of Kali Linux

  • 2004: BackTrack Linux was released - the predecessor to Kali
  • 2013: Kali Linux 1.0 launched, replacing BackTrack
  • 2015: Kali 2.0 released with rolling updates
  • 2016-Present: Continuous development with quarterly releases
  • Key Fact: Developed and maintained by Offensive Security

What is Debian?

Debian is one of the oldest and most stable Linux distributions. It serves as the foundation for many other distributions (including Kali, Ubuntu, and Mint). Debian is known for its:

  • Strict open-source policy
  • Extensive package repositories (over 59,000 packages)
  • Robust package management system (APT)
  • Stable release cycle

Debian-Based Distributions

# Major Debian-based distributions:
- Kali Linux (Security Testing)
- Ubuntu (Desktop/Server)
- Linux Mint (Desktop)
- Parrot OS (Security/Privacy)
- Elementary OS (Desktop)
- Tails (Privacy/Anonymity)

Why Use Kali Linux?

Pre-Installed Tools

Comes with 600+ security tools for penetration testing, forensics, and reverse engineering.

Rolling Release

Continuous updates ensure you always have the latest security tools and kernel.

Custom Kernel

Patched for injection support (wireless and network).

Secure by Default

Network services disabled by default, non-root user with sudo privileges.

Important Considerations

Kali Linux is designed as a specialist tool, not a general-purpose operating system. It's optimized for security assessments and may not be suitable for daily desktop use due to its specialized configuration and security-focused defaults.

Kali Linux Boot Process

Understanding the boot process is essential for troubleshooting and forensic analysis. Here's the complete journey from power-on to login prompt:

1
BIOS / UEFI Initialization

BIOS (Basic Input/Output System) or UEFI (Unified Extensible Firmware Interface) is the first software that runs when you power on your computer.

# BIOS Responsibilities:
- Power-On Self-Test (POST)
- Initialize hardware (CPU, RAM, storage)
- Locate bootable devices
- Load bootloader from MBR (BIOS) or GPT (UEFI)

UEFI advantages: Faster boot, secure boot, support for drives >2TB, GUI interface.

2
GRUB Bootloader

GRUB (GRand Unified Bootloader) is the default bootloader for most Linux distributions including Kali.

# GRUB Configuration File:
/boot/grub/grub.cfg (auto-generated)
/etc/default/grub (user configuration)

# Regenerate GRUB config after changes:
sudo update-grub

GRUB loads the Linux kernel and initial RAM disk (initrd/initramfs) into memory and passes control to the kernel.

3
Linux Kernel Initialization

The Linux Kernel is the core of the operating system that manages hardware resources and system processes.

# Kernel Responsibilities:
- Initialize remaining hardware
- Mount the root filesystem
- Decompress initramfs
- Start the init process (PID 1)

Kali uses a patched kernel with additional drivers for wireless injection and packet forging.

4
Systemd & Service Management

Systemd is the init system and service manager that replaces the traditional SysV init. It's responsible for starting and managing system services.

# Systemd key commands:
sudo systemctl start service_name
sudo systemctl stop service_name
sudo systemctl enable service_name
sudo systemctl status service_name

# Check boot time:
systemd-analyze blame

Systemd starts services in parallel, significantly improving boot times compared to sequential SysV init.

5
Login & Desktop Environment

Finally, the system starts the display manager (usually LightDM or GDM) which presents the login screen.

# Default Kali login:
Username: kali
Password: kali

# Root access:
sudo su -
# or
sudo -i

After authentication, the desktop environment (Xfce by default) starts, and you have a fully functional Kali Linux system.

Boot Process Commands

# View kernel messages from current boot:
dmesg | less

# View system boot log:
journalctl -b

# Check init system:
ps -p 1 -o comm=

# List available kernels:
ls -la /boot/

Linux Package Manager

What is a Package Manager?

A package manager is a collection of software tools that automates the process of installing, upgrading, configuring, and removing software packages from a computer's operating system.

Key Functions

  • Dependency Resolution: Automatically installs required libraries
  • Version Management: Handles software versions and updates
  • Repository Management: Connects to online software repositories
  • Verification: Checks package integrity with cryptographic signatures
  • Clean Removal: Removes packages and unused dependencies

Why It's Essential

Without a package manager, you would need to:

  • Manually download software from websites
  • Resolve dependencies yourself
  • Compile from source code
  • Manually update each application
  • Risk version conflicts and security issues

APT - Advanced Package Tool

Kali Linux uses APT (Advanced Package Tool), the package manager from Debian. APT works with .deb packages and manages dependencies automatically.

# APT Command Structure:
sudo apt [command] [options] [package]

# Update package lists:
sudo apt update

# Upgrade installed packages:
sudo apt upgrade

# Full system upgrade:
sudo apt full-upgrade

# Install a package:
sudo apt install nmap

# Remove a package (keep config):
sudo apt remove nmap

# Remove package and config:
sudo apt purge nmap

# Search for packages:
apt search "penetration testing"

# Show package info:
apt show nmap

# List installed packages:
apt list --installed

Other Package Managers

YUM/DNF (RHEL/Fedora)

sudo yum install package
sudo dnf install package # newer

Pacman (Arch Linux)

sudo pacman -S package
sudo pacman -Syu # update system

Zypper (openSUSE)

sudo zypper install package
sudo zypper update

Kali Repository Warning

Kali Linux uses its own specialized repositories. Do not add Ubuntu or other Debian repositories to your Kali sources, as this will likely break your system due to package conflicts and incompatible versions.

Package Management Files

# APT configuration:
/etc/apt/sources.list # Main repository list
/etc/apt/sources.list.d/ # Additional repos
/var/lib/apt/lists/ # Downloaded package lists
/var/cache/apt/archives/ # Downloaded .deb packages

# Clean package cache:
sudo apt clean # Remove ALL cached packages
sudo apt autoclean # Remove old versions only
sudo apt autoremove # Remove unused dependencies
